# @nimpl/path-parser

<PackageLinks npmName="@nimpl/path-parser" githubName="nimpl-path-parser" />

Helper for next.js-styled pathname parsing (f.e. `/(site)/docs/[key]`) to get dynamic params.

## Installation

```bash switcher tab="npm"
npm i @nimpl/path-parser
```

```bash switcher tab="yarn"
yarn add @nimpl/path-parser
```

## Usage

```ts
import { parse } from "@nimpl/path-parser";

// ...

const params = parse("/docs/path-parser", "/(site)/docs/[key]"); // { key: 'path-parser' }
const { segments } = parse("/docs/path-parser", "/(site)/[[...segments]]"); // { segments: ['docs', 'path-parser'] }
```

## Arguments

`pathname` - clean page pathname;

`rule` - the expected rule (_same as the path to the page directory_).

<Hint>
If pathname does not match the expected rule - the parser will return `null`.
</Hint>
